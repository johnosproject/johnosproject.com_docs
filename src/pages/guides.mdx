# User Guides

import Grid from '@mui/material/Grid';
import Typography from '@mui/material/Typography';
import Paper from '@mui/material/Paper';

Each IoT solution require a perfect coordination between different actors:
* [Maker](/docs/what/josp/actors/maker): how design the connected object
* [Developer](/docs/what/josp/actors/developer): how implement the software that interact with connected objects
* [End User](/docs/what/josp/actors/end_user): how will use objects and software created by Makers and Developers

**The JOSP EcoSystem, with his tools and guidelines, allow those actors work independently.**

Remember that in a [JOSP EcoSystem](/docs/what/josp/ecosystem), the End User is free to compose his solutions.<br/>
He can look for the [JOSP Service](/docs/what/josp/service) that accomplish with his requirements *(p.e. manage smart home light, collect data from environment sensors, perform predictive maintenance analysis...)*,
then allow desired software interact with [JOSP Object](/docs/what/josp/object) *(p.e. lights, sensor devices, industrial machines)*.

**The JOSP Docs split the JOSP user's guides by actors:**
